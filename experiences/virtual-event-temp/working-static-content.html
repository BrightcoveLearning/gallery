Working with Static Content

In this topic you will learn how to add static content to a Virtual Event Experience.

Static text objects are used to inject custom HTML into the template. This sample hero banner was constructed using a combination of text and images and injected into the top of the page.

Static content can be any HTML and can be injected anywhere in the template. Static content objects are added to the static_content object in the configuration file

The Virtual Event template has four main areas where videos are displayed as designated in the sample site:

  1. Featured collection
  2. Primary collection
  3. Secondary collection
  4. Tracks

When ordering collections in Gallery, the collections should be ordered in the following manner:

  • Collection 1 - Featured collection
  • Collection 2 - Secondary collection
  • Collection 3 - Primary collection
  • Collection 4 - Tracks

Configuring event tracks

When creating the tracks, follow the guidelines below to make sure they appear correctly in the experience:

  • The tracks collection should be the fourth collection
  • The collection name will appear in the virtual event template so use a meaningful name
  • The tracks collection will be a nested collection, no videos will appear in this collection
  • Create child collections under the tracks collection for each track
  • Assign videos to the child collections, remember, do not use playlists when creating the collections

Configuring the tracks URL

Once the collections have been created, the configuration file needs to be updated so that the On-Demand Sessions link in the header will direct to the correct URL. The name used for the tracks collection (collection 4) will be used in the URL to access the tracks page. To ensure the URL is set correctly, confirm the following.

  1. Review the configuration file and locate the breakout_collection object.
  2. The collection parameter contains the URL of the tracks page. For example:
    
"breakout_collection": {
    "collection": "/category/videos/tracks",
    "insert_target": ".bc-on-demand-section",
    "insert_mode": "after"
},
  3. The last path in the collection URL should reflect the name of your tracks collection (collection 4). For example, if your tracks collection (collection 4) is named Sessions, the collection parameter would be updated to:
    
"collection": "/category/videos/sessions",
  4. To verify the configuration, click the On-Demand Sessions link in the header. The tracks session should appear. Clicking on a track should open the videos in a track. If you see a 404 error, there is a configuration issue.
