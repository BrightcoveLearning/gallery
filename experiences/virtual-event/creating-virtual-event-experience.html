Creating a Virtual Event Experience

In this topic you will learn about creating a Virtual Event Experience.

Gallery provides a Virtual Event Portal template which can be used to create Virtual Event Experiences for hosting and delivering virtual events. After creating a new Virtual Event Experience, you will be prompted to pre-populate the experience. Brightcove recommends choosing No thanks.

Much of the experience customization is done using a configuration file. The configuration file is added to the template as part of the experience header. After creating a new Virtual Event Portal Experience, the first thing that should be done is to add the configuration file to the template.

The configuration file controls some of the general template options and is also used to:

  • Customize the header
  • Customize the footer
  • Configure the event schedule
  • Customize the event speakers
  • Customize the event sponsors

The process to add videos to the experience is similar to other Gallery Portal Experiences although the Virtual Event template does expect the collections to be organized in a specific way. For information on adding videos, see Adding Videos to a Virtual Event Experience.

Adding the configuration file to the template

The Virtual Event Portal Experience requires a configuration file be included with the template. The configuration file includes site settings, layout options, event schedule, speaers and other content that will be displayed. After creating a new Virtual Event Experience, adding the config file is the first thing that should be done. For information on how to add the configuration file, see Adding the Configuration File to the Template.

Configuring the header

The header displays at the top of all the pages and provides a way to brand and add social icons to the site. For more information on configuring the header, see Customizing the Header for a Virtual Event Experience.

Configuring the footer

The footer displays at the bottom of all the pages and provides a way to brand and add social icons and links to the site. For more information on configuring the footer, see Configuring the Footer for a Virtual Event Experience.

Creating the post-event state

The post-event state is when the live event has finished. The goal of the post-event state is to encourage site visitors to engage more with the brand, provide an on demand version of the live event and provide links to related content.

For more information on creating the post-event state, see Creating the Post-Event State for a Portal Event Experience.

Setting the home page state

Once the pre-, live and post-event state pages have been created, the Gallery site can be configured so the home page displays the appropriate state. To change the home page state, follow these steps.

  1. Edit the Live Event Portal Experience.
  2. The current home page state will be displayed in the left navigation.
  3. Click Change. The Change Home Page dialog will open.
  4. Select a home page state and then click Change.

The experience MUST be republished after the home page state is changed. Note that it may take a couple of minutes for the state change to be reflected on the live site.

Note: You, the publisher, are responsible for changing the home page state. Gallery will never automatically switch the home page state.
